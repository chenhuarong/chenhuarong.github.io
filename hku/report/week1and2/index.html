Observations & Thoughts

About the input

As illustrated above, the input of PaintTween is a combination of many features, which means that it will be hard to extend the dataset.

The input contains the image, orientation, velocity(forward and backward), distance transform, etc. The image is necessary to stylized image synthesizing and other input terms are mainly contributed to temporal coherence in video synthesizing.

About the algorithm efficiency

The stylization problem is a complex problem and a large amount of computation is required. Take PaintTween as an example, it takes 0.5 to 1 minute to synthesize a 960*540 stylized image and about 2 hours to synthesize a 100-frame video in my laptop. See the table below for statistics about time consumption.

About my understanding of stylization problem

For not looking so much trouble, I think I can start with stylized image synthesizing and extend it to video synthesizing later. And I can choose common image formats like jpg, png, bmp as the image input.

Limited by my hardware, I should strike a balance between effectiveness and efficiency in my implementation of stylization.

Back to the problem, we want to synthesize a stylized image \(I'\) from source image \(I\)(\(M*N\)). If we have a huge database storing all the corresponding relations between source and stylized image of size \(M*N\), what we should do is just find out the relation. But it is impossible because there are non-enumerative possible sources. Then what we should do is to use the information from example images pair(the style input and output) as much as possible to “copy” the style to the synthesized image.

Gaussian Pyramid and Coarse-to-Fine methods are widely used in previous works. Practice has proved they are effective. PatchMatch(Link) is also used in recent works for a higher synthesis efficiency.

I am wondering if it is possible to synthesize the “most like” part before synthesizing the “not so like” part to improve the existing algorithm. For example, look at the picture below, the big patch \(A\) in \(S\) and \(B\) in \(I\) is very close to each other(we can choose some features to quantify “close”). If we synthesize the big patch area first, it may save time and have contribution to synthesis in adjacent areas.

This idea can also combine with Gaussian Pyramid and Coarse-to-Fine methods, and PatchMatch may be helpful to find the best match of given “big patch”.
